WikiRag 调试台

这个页面按真实链路调试：先搜 Wiki，再导入 source，再做 normalize，最后发起 RAG 查询。 页面不是直接调用 Huawei Wiki，而是调用你自己的后端接口；后端导入 Wiki 时会在 /api/source/import 里触发 https://wiki.huawei.com/devops-knowledge-management/api/getWiki。

  • 当前链路 /api/wiki/search -> /api/source/import(fetch_from_wiki=true) -> getWiki -> /api/knowledge/normalize -> /api/rag/query
  • 你要确认的点 是否能搜到 Wiki、是否能导入正文、是否真的拿到 raw_html/rendered_text、是否能进入 RAG 命中。
  • API Key 当前页面所有请求都会带 X-API-Key。

步骤 1：全局参数

步骤 2：Wiki 搜索与导入

如果你的目标是“根据个人背景推荐值得阅读的 Wiki”，先点击下面的“生成推荐 Wiki”，再从推荐结果里选中条目导入。
推荐接口会根据你的背景文本抽关键词、生成搜索词，并聚合 Huawei Wiki 搜索结果。它不会自动导入。

先搜，再选，再导入。导入动作会触发后端调用 getWiki 详情接口。

选中某条结果后，点击“导入选中 Wiki”即可。导入使用 fetch_from_wiki=true，后端会调用 getWiki。

步骤 3：规范化与 RAG 查询

如果你已经通过上一步导入了 source，可以直接点击“规范化当前 source”。规范化完成后，再发起 RAG 查询。
