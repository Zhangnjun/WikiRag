WikiRag 双场景工作台

当前版本把能力拆成两个入口：一个服务运维助手方向的知识库构建与 RAG 调试，一个服务个人求职方向的定向搜索与业务细节提炼。 `/rag-debug` 现在作为统一入口，避免把所有流程继续堆在一个页面里。

运维知识库 / RAG 工作台

面向内部运维助手和知识库建设场景，重点是搜 Wiki、导入 source、规范化入库、RAG 查询与链路调试。

  • 搜索并导入 Huawei Wiki
  • 构建 knowledge / chunk / embedding
  • 发起 RAG 查询并查看引用
  • 验证运维知识沉淀链路
进入运维工作台 查看接口文档

个人搜索 / 求职细节提取工作台

面向个人背景增强和项目补充场景，重点是根据简历与目标方向扩展关键词，挖出适合写进项目经验的真实业务证据。

  • 输入简历背景和目标方向
  • 自动扩展 RAG / Agent / 微调 / 推理等相关词
  • 输出推荐 Wiki、项目证据和 skill 可行性
  • 支持导出推荐 Excel
